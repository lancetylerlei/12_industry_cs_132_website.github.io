Data Description

This dataset is a continuation of the PH FinTech Apps Google Store Reviews (2023), which was originally gathered using RAPIDAPI. In this updated version, data was collected using GoogleScraper, allowing for a more comprehensive and structured extraction of user reviews from the Google Play Store. With 65,535 unique reviews, this dataset provides a robust source for analyzing public perception, user satisfaction, and fintech app performance over time.

The dataset contains various attributes that capture key aspects of user feedback. Each review entry includes a unique review identifier (reviewId) and a timestamp (review_datime_utc) to track when the review was submitted. Additionally, user information such as author name (author_name) and author identifier (author_name_guid) is included, ensuring that repeated reviews from the same user can be analyzed while maintaining data privacy.

To assess user sentiment and engagement, the dataset records the review text (review_text), the rating (review_rating) on a scale of 1 to 5, and the number of likes (review_likes) a review has received from other users. These features allow for deeper analysis of how user feedback evolves and how others perceive the value of certain reviews.

In terms of app-related metadata, each review is linked to a specific fintech application (application_id) and the app version (author_app_version) at the time of the review. This allows researchers to evaluate whether updates lead to higher satisfaction and improved ratings over time.

The dataset serves multiple purposes, making it an essential tool for fintech developers, data analysts, and researchers. It enables sentiment analysis to track how users perceive fintech apps, trend analysis to evaluate the impact of app updates, and topic modeling to identify key pain points in user feedback. Moreover, it allows comparisons between different fintech developers, highlighting the best and worst-performing apps based on user engagement and satisfaction.

This dataset is particularly useful for longitudinal studies, as it is continuously updated to reflect real-time changes in user sentiment. By leveraging this data, fintech companies can refine their app development strategies, address user concerns more effectively, and enhance the overall digital financial experience. With its structured format and rich content, this dataset is a valuable resource for understanding the evolving landscape of fintech applications in the Philippines.

Data Preprocessing

To ensure data accuracy and usability, preprocessing involved several key steps:

  • Data Cleaning: Removed duplicates, standardized fields, and formatted timestamps.
  • Text Preparation: Lowercased text, removed stopwords, stripped special characters, and flagged spam reviews.
  • Feature Engineering: Generated sentiment scores, review length bins, keyword tags, and app version mapping.
  • Outlier Detection: Identified extreme ratings and unusual like counts.
  • Time-Series Formatting: Converted UTC timestamps to PHT for consistency.

Data Sampling and Filtering

To ensure data accuracy and usability, data sampling and filtering involved several key steps:

  • Review Translation: Translated all reviews from Filipino to English.
  • Filling Missing Values: Filled missing review texts with "ok" to be removed by "constructive" review model. (See Google Colab)
  • "Constructive" Reviews Filtering: Filtered all reviews with a "constructive" score of >= 0.8.
  • "Constructive" Reviews Sampling: 7000 "constructive" reviews were randomly taken.
  • Manually Filtering and Classification: Manually removed unaccounted, unconstructive reviews and manually classified 6566 remaining reviews.

This robust preprocessing ensured a high-quality dataset, ready for exploratory data analysis and machine learning analysis.

Machine Learning & SHAP Analysis

We used XGBoost and SHAP to determine which review categories most significantly influence Google Play Store ratings for fintech apps. These tools were selected for their scalability and interpretability, making them ideal for large-scale, explainable analysis.

XGBoost handles non-linear relationships well and outperforms traditional decision trees on large datasets. SHAP (SHapley Additive exPlanations) complements it by identifying which features push predictions higher or lower—helping us understand what truly drives positive and negative app reviews.

Discussion

According to the SHAP bar and beeswarm plots, user friendliness and transaction speed are the two most significant features impacting review scores.

User friendliness had the lowest mean SHAP value at -0.052496 but the highest mean absolute SHAP value of 0.634410, meaning it strongly affects ratings and is often a pain point. This suggests many apps underperform in usability, leading to lower reviews.

Transaction speed, with a positive mean SHAP of 0.017774, indicates that users appreciate apps that "just work" and perform actions quickly. This feature positively correlates with higher ratings.

Another insight: loading speed (mean absolute SHAP = 0.065434, mean SHAP = -0.025441) is the seventh most important feature but contributes to lower ratings. It's closely tied to user friendliness and confirms that app responsiveness remains a common frustration.

Conclusion

In conclusion, user friendliness and transaction speed are the top drivers of fintech app satisfaction. However, many apps struggle with delivering a seamless user experience—highlighting a clear area for improvement.

App developers are encouraged to focus on streamlining UI/UX, improving responsiveness, and reducing loading times. Investing in usability and performance will not only boost ratings but build long-term trust and engagement with users.

Future research should investigate subcomponents of these key factors, such as specific UI elements, task flows, or common bugs, to pinpoint more targeted recommendations for development teams.

Our goal was to transform messy user reviews into actionable product insights—and we did so through data-driven tools that explain themselves.
